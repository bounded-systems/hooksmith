# CLI-Based Hooks System

This document describes the comprehensive hooks system where the Rust CLI becomes the single source of truth for all validation logic, with Lefthook serving as a dumb runner.

## Overview

The hooks system provides:
- **Auto-installation**: Hooks are automatically installed on first CLI usage
- **Single source of truth**: All validation logic is in the Rust CLI
- **Zero duplication**: Same checks run locally and in CI
- **Comprehensive validation**: Format, lint, tests, WIT validation, safety checks

## Architecture

### 1. CLI as Single Source of Truth

All validation logic is implemented in Rust within the CLI:

```rust
// src/commands/hooks.rs
pub async fn pre_commit_checks(config: &CliConfig) -> Result<CliResult> {
    // 1. Format check
    run_command("cargo", &["fmt", "--all", "--", "--check"])?;
    
    // 2. Lint check
    run_command("cargo", &["clippy", "--workspace", "--all-targets", "--", "-D", "warnings"])?;
    
    // 3. Tests
    run_command("cargo", &["test", "--workspace"])?;
    
    // 4. WIT validation
    run_command("cargo", &["component", "build", "--release"])?;
    
    // 5. Worktree safety
    run_worktree_guard()?;
    
    // 6. File naming check
    run_main_files_check()?;
    
    Ok(CliResult::success("All pre-commit checks passed"))
}
```

### 2. Lefthook as Dumb Runner

Lefthook configuration is simplified to just call the CLI:

```yaml
# lefthook.yml
pre-commit:
  commands:
    cli-pre-commit:
      run: cargo run --bin pushd-cli -- hooks pre-commit
      glob: "*"

pre-push:
  commands:
    cli-pre-push:
      run: cargo run --bin pushd-cli -- hooks pre-push
```

### 3. Auto-Installation

Hooks are automatically installed when the CLI is first used:

```rust
// src/cli_main.rs
async fn ensure_hooks_installed() {
    if !std::path::Path::new("lefthook.yml").exists() {
        print_info("📌 Auto-installing Lefthook hooks...");
        
        let result = std::process::Command::new("cargo")
            .args(&["run", "--bin", "pushd-cli", "--", "hooks", "install"])
            .status();
        
        // Handle result...
    }
}
```

## Validation Logic

### Pre-Commit Checks

1. **Code Formatting**: `cargo fmt --all -- --check`
2. **Linting**: `cargo clippy --workspace --all-targets -- -D warnings`
3. **Tests**: `cargo test --workspace`
4. **WIT Validation**: `cargo component build --release`
5. **Worktree Safety**: Custom worktree guard script
6. **File Naming**: Prevent files named "main"

### Pre-Push Checks

1. **Integration Tests**: `cargo test --package pushd-cli --test integration`
2. **Documentation Generation**: `cargo run --bin pushd-cli -- docs generate`
3. **CLI Validation**: `cargo run --bin pushd-cli -- validate all`
4. **Push Safety**: Prevent pushing to main from CLI prototype

### Commit Message Validation

- Uses `commitlint` for conventional commit validation
- Gracefully handles missing `commitlint` installation

### Post-Commit Hooks

1. **Worktree Status Update**: Update worktree tracking
2. **Documentation Update**: Regenerate CLI documentation

### Post-Checkout Hooks

1. **Worktree Setup**: Configure environment for new worktree

## Auto-Installation Process

### 1. Detection

The CLI automatically detects when hooks are missing:

```rust
// Check if we're in a Git repository
let output = std::process::Command::new("git")
    .args(&["rev-parse", "--git-dir"])
    .output();

// Check if lefthook.yml exists
if !std::path::Path::new("lefthook.yml").exists() {
    // Auto-install hooks
}
```

### 2. Installation

When hooks are missing, the CLI:

1. **Generates lefthook.yml**: Creates configuration file
2. **Checks Lefthook**: Verifies Lefthook is installed
3. **Runs lefthook install**: Installs Git hooks
4. **Validates installation**: Confirms hooks are working

### 3. Configuration Generation

The CLI generates a complete `lefthook.yml`:

```yaml
# Auto-generated by pushd-cli
pre-commit:
  commands:
    cli-pre-commit:
      run: cargo run --bin pushd-cli -- hooks pre-commit
      glob: "*"

pre-push:
  commands:
    cli-pre-push:
      run: cargo run --bin pushd-cli -- hooks pre-push
# ... more hooks
```

## Safety Features

### 1. Worktree Protection

- Prevents direct `git checkout -b` usage
- Enforces CLI usage: `pushd-cli worktree create <name>`
- Validates worktree setup

### 2. File Naming Protection

- Prevents creating files named "main"
- Avoids confusion with main branch
- Suggests alternative names

### 3. Push Protection

- Prevents pushing CLI prototype to main
- Enforces proper branch protection
- Provides clear error messages

### 4. Component Validation

- Validates WIT interfaces
- Ensures components build correctly
- Checks component structure

## Error Handling


## Troubleshooting

### Common Issues

1. **Lefthook not installed**
   ```bash
   # Install Lefthook
   brew install lefthook  # macOS
   # or
   curl -1sLf 'https://dl.cloudsmith.io/public/evilmartians/lefthook/setup.deb.sh' | sudo -E bash
   sudo apt-get update && sudo apt-get install lefthook  # Ubuntu
   ```

2. **CLI not built**
   ```bash
   # Build the CLI first
   cargo build --release --package pushd-cli
   ```

3. **Hooks not working**
   ```bash
   # Reinstall hooks
   pushd-cli hooks install
   ```

### Debug Mode

Enable debug output for troubleshooting:

```bash
# Run with debug logging
RUST_LOG=debug cargo run --bin pushd-cli -- hooks pre-commit
```
