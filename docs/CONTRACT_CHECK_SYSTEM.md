# 🔗 Hooksmith Contract Check System

The **Contract Check System** is a comprehensive validation pipeline that ensures the Hooksmith project maintains its goal of **100% Rust-owned file generation**. This system provides a single entry point for all validation and status checks, making it easy to integrate into CI/CD pipelines and local development workflows.

## 🎯 Overview

The contract check system validates:

1. **Generated Files**: Ensures no manually modified generated files
2. **Migration Progress**: Tracks progress toward 100% Rust-owned coverage
3. **File Type Budget**: Enforces the ≤8 file extension limit
4. **Trend Tracking**: Monitors progress over time

## 🚀 Quick Start

### Basic Usage

```bash
# Run all checks (non-strict mode)
cargo xtask contract-check

# Run with strict validation (fails on violations)
cargo xtask contract-check --strict

# Run with detailed output
cargo xtask contract-check --verbose

# Generate trend data
cargo xtask contract-check --trend
```

### Command Options

```bash
cargo xtask contract-check [OPTIONS]

Options:
  --staged-only              Check only staged files
  --strict                   Exit with error on violations
  --trend                    Generate trend data
  --trend-output <DIR>       Output directory for trend data [default: status-trends]
  --verbose                  Show detailed output
  -h, --help                 Print help
```

## 📊 What Gets Checked

### 1. Generated Files Validation

Validates that all files marked as `linguist-generated=true` in `.gitattributes` are actually generated by the Rust pipeline and haven't been manually modified.

**Checks:**
- Files with `linguist-generated=true` have proper generation markers
- No manual modifications to generated files
- All generated files have valid checksums

**Example Output:**
```
1️⃣ Validating generated files...
✅ All generated files are valid!
```

### 2. Migration Progress Check

Tracks progress toward the goal of 100% Rust-owned coverage (Rust files + generated files).

**Checks:**
- Current migration progress percentage
- File type breakdown and categorization
- Migration recommendations

**Example Output:**
```
2️⃣ Checking migration progress...
⚠️  Migration progress below goal: 7.1% (target: 95.0%)
```

### 3. File Type Analysis

Provides comprehensive breakdown of all file types in the repository.

**Information:**
- File extension counts
- Migration status (Keep/Generate/Remove/Consolidate)
- Priority and effort estimates
- Target actions

**Example Output:**
```
4️⃣ File type analysis...
📁 Hooksmith File Type Analysis
===============================
[
  {
    "extension": "rs",
    "count": 107,
    "is_generated": true,
    "migration_status": "Keep",
    "target_action": "Keep as Rust source",
    "priority": 1,
    "estimated_effort": "N/A"
  }
]
```

### 4. Trend Data Generation (Optional)

Tracks progress over time by saving snapshots of the current state.

**Features:**
- Daily/weekly snapshots
- Progress visualization data
- Historical trend analysis


```yaml
- name: Run contract check

  run: cargo xtask status migration-progress --format markdown > status.md
```

### Pre-commit Hooks

```bash
#!/bin/bash
# .git/hooks/pre-commit

echo "🔗 Running contract check..."
cargo xtask contract-check --staged-only --strict

if [ $? -ne 0 ]; then
    echo "❌ Contract check failed!"
    echo "Please fix the issues above before committing."
    exit 1
fi

echo "✅ Contract check passed!"
```

## 🎯 Goals and Targets

### Migration Progress Goals

- **Target**: 95% migration progress
- **Current**: 7.1% (as of latest check)
- **Calculation**: (Approved types / Total types) × 100

### File Type Budget

- **Target**: ≤8 total file extensions
- **Current**: 14 file types
- **Strategy**: Consolidate and remove unnecessary types

### Coverage Goals

- **Target**: 100% Rust-owned coverage
- **Components**: Rust files + Generated files
- **Exclusions**: Manual files that can't be generated

## 🔍 Troubleshooting

### Common Issues

1. **Generated Files Modified**
   ```
   ❌ Generated files validation failed: File 'docs/README.md' has been manually modified
   ```
   **Solution**: Regenerate the file using `cargo xtask gen-docs`

2. **Migration Progress Low**
   ```
   ⚠️  Migration progress below goal: 7.1% (target: 95.0%)
   ```
   **Solution**: Migrate file types according to priority order

3. **File Type Budget Exceeded**
   ```
   ❌ File type budget exceeded: 14 types (limit: 8)
   ```
   **Solution**: Consolidate similar file types or remove unnecessary ones

## 🚀 Advanced Usage

### Custom Validation Rules

You can extend the contract check system by adding custom validation rules:

```rust
// In xtask/src/main.rs
async fn run_contract_check(...) -> Result<()> {
    // ... existing checks ...
    
    // Custom validation
    if let Err(e) = custom_validation().await {
        errors.push(format!("   ❌ Custom validation failed: {}", e));
        if strict {
            all_passed = false;
        }
    }
    
    // ... rest of function ...
}
```

## 🤝 Contributing

When contributing to the contract check system:

1. **Add new checks** in the `run_contract_check` function
2. **Update documentation** for new features
3. **Add tests** for validation logic
4. **Consider CI integration** for new checks

### Example: Adding a New Check

```rust
// Step 5: Custom validation
println!("\n5️⃣ Custom validation...");
match custom_validation().await {
    Ok(_) => {
        println!("   ✅ Custom validation passed");
    }
    Err(e) => {
        let error_msg = format!("   ❌ Custom validation failed: {}", e);
        errors.push(error_msg.clone());
        if strict {
            all_passed = false;
        }
        if verbose {
            println!("{}", error_msg);
        }
    }
}
```
