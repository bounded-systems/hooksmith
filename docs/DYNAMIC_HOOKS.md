# Dynamic Hooks Generation System

This document describes the advanced dynamic hooks generation system that automatically creates Lefthook configurations based on project structure and available features.

## Overview

The dynamic hooks system provides:
- **Project-aware configuration**: Hooks are generated based on actual project structure
- **Feature detection**: Automatically detects Rust, WASM, Docker, daemons, etc.
- **Auto-updates**: Hooks update when project structure changes
- **Zero configuration**: No manual lefthook.yml editing required
- **Comprehensive coverage**: All project features get appropriate hooks

## How It Works

### 1. Project Feature Analysis

The CLI analyzes the project structure to detect available features:

```rust
#[derive(Debug, Default)]
struct ProjectFeatures {
    has_rust_workspace: bool,
    has_wasm_components: bool,
    has_wit_interfaces: bool,
    has_integration_tests: bool,
    has_daemons: bool,
    has_docker: bool,
    has_worktree_management: bool,
    has_safety_checks: bool,
    has_documentation: bool,
    has_commitlint: bool,
}
```

### 2. Dynamic Configuration Generation

Based on detected features, the CLI generates appropriate hooks:

```yaml
# Auto-generated by pushd-cli -- hooks generate
# Features detected:
# - Rust workspace: true
# - WASM components: true
# - WIT interfaces: true
# - Integration tests: true
# - Daemons: true
# - Docker: true
# - Worktree management: true
# - Safety checks: true
# - Documentation: true
# - Commitlint: true

pre-commit:
  parallel: true
  commands:
    # CLI-based pre-commit checks (single source of truth)
    cli-pre-commit:
      run: cargo run --bin pushd-cli -- hooks pre-commit
      glob: "*"

    # Rust-specific hooks
    format:
      run: cargo fmt --all -- --check
      glob: "*.rs"
      stage_fixed: true

    lint:
      run: cargo clippy --workspace --all-targets -- -D warnings
      glob: "*.rs"

    # WIT interface validation
    wit-validate:
      run: cargo component build --release
      glob: "wit/*.wit"

    # Safety checks
    worktree-guard:
      run: ./scripts/git/hooks/worktree_guard.sh
      glob: "*"

    prevent-main-files:
      run: ./scripts/git/hooks/prevent_main_files.sh
      glob: "*"

    # Daemon-specific hooks
    daemon-validation:
      run: cargo run --bin pushd-cli -- validate daemons
      glob: "daemon/*.rb"

    # Docker-specific hooks
    docker-validation:
      run: cargo run --bin pushd-cli -- docker validate
      glob: "docker-compose*.yml"

pre-push:
  commands:
    # CLI-based pre-push checks
    cli-pre-push:
      run: cargo run --bin pushd-cli -- hooks pre-push

    # Integration tests
    integration-tests:
      run: cargo test --package pushd-cli --test integration

    # Documentation generation
    generate-docs:
      run: cargo run --bin pushd-cli -- docs generate

commit-msg:
  commands:
    # CLI-based commit message validation
    cli-commit-msg:
      run: cargo run --bin pushd-cli -- hooks commit-msg $1

post-commit:
  commands:
    # CLI-based post-commit hooks
    cli-post-commit:
      run: cargo run --bin pushd-cli -- hooks post-commit

post-checkout:
  commands:
    # CLI-based post-checkout hooks
    cli-post-checkout:
      run: cargo run --bin pushd-cli -- hooks post-checkout {prev} {curr} {flag}
```

## Feature Detection

### Rust Workspace Detection

```rust
// Check for Rust workspace
let cargo_toml = repo_root.join("Cargo.toml");
if cargo_toml.exists() {
    features.has_rust_workspace = true;
    
    // Check for specific components
    let components_dir = repo_root.join("components");
    if components_dir.exists() {
        features.has_wasm_components = true;
        
        // Check for WIT files
        let wit_dir = repo_root.join("wit");
        if wit_dir.exists() {
            features.has_wit_interfaces = true;
        }
    }
    
    // Check for integration tests
    let tests_dir = repo_root.join("tests");
    if tests_dir.exists() {
        features.has_integration_tests = true;
    }
}
```

### Daemon Detection

```rust
// Check for daemon files
let daemon_dir = repo_root.join("daemon");
if daemon_dir.exists() {
    features.has_daemons = true;
}
```

### Docker Detection

```rust
// Check for Docker files
let docker_compose = repo_root.join("docker-compose.yml");
let dockerfile = repo_root.join("Dockerfile");
if docker_compose.exists() || dockerfile.exists() {
    features.has_docker = true;
}
```

### Worktree Management Detection

```rust
// Check for worktree scripts
let worktree_scripts = [
    "scripts/git/hooks/worktree_guard.sh",
    "scripts/git/hooks/worktree_setup.sh",
    "scripts/git/hooks/worktree_status.sh",
];

for script in &worktree_scripts {
    if repo_root.join(script).exists() {
        features.has_worktree_management = true;
        break;
    }
}
```

### Safety Checks Detection

```rust
// Check for safety scripts

    "scripts/git/hooks/prevent_main_push.sh",

];

for script in &safety_scripts {

        features.has_safety_checks = true;
        break;
    }
}
```

## Auto-Update System

### Project Change Detection

The CLI automatically detects when project structure has changed:

```rust
// Check if project structure has changed
let project_files = [
    "Cargo.toml",
    "components/",
    "wit/",
    "daemon/",
    "docker-compose.yml",
    "Dockerfile",
    "tests/",
    "docs/",
];

let mut has_changes = false;
for file in &project_files {
    if std::path::Path::new(file).exists() {
        // Check if this file/directory has been modified recently
        let git_status = std::process::Command::new("git")
            .args(&["status", "--porcelain", file])
            .output();
        
        if let Ok(output) = git_status {
            if !output.stdout.is_empty() {
                has_changes = true;
                break;
            }
        }
    }
}
```

### Automatic Updates

When changes are detected, hooks are automatically updated:

```bash

# Output: 🔄 Auto-updating hooks configuration...
# Output: ✅ Hooks auto-updated successfully
```
