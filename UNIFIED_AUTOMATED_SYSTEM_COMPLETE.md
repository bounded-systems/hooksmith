## 🏗️ **System Architecture**

### **Source of Truth**: `generated-sources/*.jsonc`
- **`lefthook-config.jsonc`** → generates `lefthook.yml`
- **`readme.jsonc`** → generates `README.md`
- **`schema.jsonc`** → defines the JSONC schema structure

### **Registry**: `config/generated-files.jsonc`
- Central record of all generated files
- Contains paths, slugs, extensions, and SHA256 checksums
- Automatically updated by the unified generator

### **Validation Pipeline**: Multiple layers of enforcement
1. **Unified Generator** - Reads JSONC specs, generates files with headers/checksums
2. **Enhanced Bootstrap** - Builds xtask, cleans files, regenerates, validates
3. **Regeneration Check** - Proves deterministic generation with zero diff guarantee
4. **Enhanced Validate Files** - Enforces strict extension policy with helpful suggestions
5. **Pre-add Git Hook** - Blocks invalid files from staging with detailed feedback
6. **CI Integration** - GitHub Actions workflow for automated validation

## 🚀 **Available Commands**

### **Core Generation**
```bash
# Generate all files from JSONC sources
cargo run -p xtask -- gen-all-unified --validate

# Clean all generated files
cargo run -p xtask -- clean-generated --verbose

# Regeneration consistency check
cargo run -p xtask -- regen-check --strict --verbose
```

### **Enhanced Bootstrap**
```bash
# Full bootstrap with cleaning and validation
cargo run -p xtask -- bootstrap --clean --validate --verbose
```

### **File Validation**
```bash
# Validate all files against strict policy
cargo run -p xtask -- validate-files --strict --verbose

# Validate only staged files
cargo run -p xtask -- validate-files --staged --strict --verbose
```
